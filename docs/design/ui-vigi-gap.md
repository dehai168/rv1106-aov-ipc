# Web UI 功能差异对照（VIGI 模拟器 vs 本板）

> 参考：[VIGI IPC Emulator](https://emulator.tp-link.com/vigi-ipc/admin/Index.htm) + [Wired Camera Web Interface User Guide](https://static.tp-link.com/upload/manual/2025/202502/20250207/1910013895_VIGI%20IPC%20Wired%20UG.pdf)  
> 目标：用 **Element Plus** 重做前端信息架构与交互，功能对齐 VIGI Web（**排除全部 PTZ**）；仅落地 **Luckfox Pico Max / 当前软件栈能支持** 的能力。  
> 状态约定：`已有` / `可做` / `部分` / `不做` / `二期`

## 1. 顶层信息架构对照

| VIGI 顶栏 | 本板现状 | 改造目标 | 判定 |
|---|---|---|---|
| Preview（实时预览） | `/preview` 有 | 保留并增强工具条 | 可做 |
| Information（信息） | 散落在系统页 | 独立「信息」：设备信息 + 日志 | 可做 |
| Settings（设置） | 扁平侧栏六页 | 左侧树：摄像机 / 事件 / 存储 / 网络 / 系统 | 可做 |
| Admin / Help / Logout | 改密+退出 | 顶栏用户菜单：改密/退出；Help 可链到 README | 可做 |
| PTZ 相关控件 | 无硬件 | **不做** | 不做 |

```
   │  ├─ 录像计划
   │  ├─ 存储管理

```

## 3. 与「必须做」边界（本里程碑）

### 3.1 纳入 M7（Element Plus + VIGI 风格改造）

1. 工程：引入 Element Plus（按需导入）、中文 locale、布局壳（顶栏+侧栏树）
2. 预览增强：比例/全屏/手动抓图；（可选）主/子切换
3. 信息：设备信息页 + 日志页
4. 摄像机：图像+码流用 Element 表单；参数分组
5. 事件：运动侦测（画框区域、7×24 布防、联动勾选）+ 告警表
6. 存储：录像计划（事件/定时）、存储管理、回放列表增强
7. 网络/系统：现有能力重做 UI；暴露预录/延录/回收水位；设备名可改

### 3.2 明确不做（本阶段）

- 全部 PTZ / 电动镜头 / 声光主动防御 / 音频对讲
- Smart Event 全家桶、LPR、邮件 FTP ONVIF 等平台接入
- 真 mkfs、在线固件升级（可留「即将推出」占位）

### 3.3 体积与性能约束

- 板端 `www/` 仍建议 gzip 后可控；Element Plus **必须按需导入**，禁止全量 CSS+全量组件
- 目标：未压缩 `www/` **&lt; 1.5MB**（现约 130KB；加 EP 后放宽但仍需监控）

## 4. 后端缺口（为 UI 改造需补的 API）

| API / 能力 | 用途 | 优先级 |
|---|---|---|
| `POST /api/v1/preview/snapshot` 手动抓图 | 预览工具条 | P0 |
| `detect.region` 支持最多 4 区（数组） | 画框多区域 | P0 |
| `record.mode` = `motion` \| `continuous` \| `schedule` + 周计划格子 | 录像计划 | P0 |
| `GET/POST record` 高级：`pre_record_sec` / `quiet_sec` / recycle% | 存储高级 | P0 |
| `GET /api/v1/system/device` 可写 `device_name` | 基本设置 | P1 |
| `GET /api/v1/system/log?tail=N` 或分页 | 日志页 | P1 |
| `POST /api/v1/system/config/export\|import` | 配置备份 | P2 |
| 告警 WS 推送 | 实时角标 | P2 |
| 主码流预览通道 | 预览切流 | P2 |
